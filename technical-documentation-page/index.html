Introduction

In the world of DevOps and system administration, automating repetitive tasks is key to saving time and reducing errors. Thatâ€™s where shell scripting comes in. It allows you to write a series of commands in a file and run them all at once, just like a program.

A shell script is a file containing a series of commands for the shell to execute. The shell itself is a command-line interpreter (CLI), while a shell script is a saved list of instructions (usually with .sh extension) like myscript.sh.

Here are the main reasons shell scripts are widely used in IT and DevOps:

  • Automate repetitive tasks as file cleanup, backups and server updates.
  • Reliable and consistent. Shell scripts run the exact same steps every time, without mistakes.
  • Native support on Unix/Linux systems, so no additional installations.
  • In DevOps, they are widely used to speed up deployments and manage infrastructure without doing everything manually.
  • They are interpreted, so there is no compile step.
Example script

Assume we create a test.sh script. Before you add anything else to your script, you need to alert the system that a shell script is being started. This is done using the shebang construct.

#!/bin/sh

To create a script containing commands, you put the shebang line first and then add the commands.

#!/bin/sh
pwd
ls

Save the above content and make the script executable.

$chmod +x myscript.sh

Now, let's run the script

./myscript.sh
Shell variables

The name of a variable can contain only letters, numbers or the underscore character.

By convention, Unix shell variables will have their names in UPPERCASE.

#!/bin/sh
NAME="John Doe"
echo $NAME
# output
$ John Doe
Special variables

Some of the special variables that you can use in your shell scripts:

  • The filename of the current script: $0
  • The number of arguments supplied to a script: $#
  • The exit status of the last command executed: $?
  • The process number of the last background command: $!
#!/bin/sh
echo "File Name: $0"
echo "First Parameter : $1"
echo "First Parameter : $1"
echo "Total Number of Parameters : $#"
